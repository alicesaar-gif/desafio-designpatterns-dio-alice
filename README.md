# 🚀 Projeto Fusion: Consolidação Inteligente de Dados para M&A

Este projeto foi desenhado sob a ótica de uma **Consultoria de Integração Tecnológica**. Em cenários de fusões e aquisições (M&A), o seu código não é apenas um "filtro de países", ele é uma ferramenta estratégica de **Consolidação de Dados**.

## 🚀 Proposta de Valor: Inovação em M&A (Mergers & Acquisitions)
Em processos de M&A, o maior desafio é unir empresas que usam sistemas diferentes. Este projeto resolve exatamente isso: ele atua como uma camada de **Inovação e Consultoria Digital**, funcionando como um **Middleware inteligente**.

### 💡 O Propósito
O Projeto Fusion resolve a "dor de cabeça" de diretores financeiros (**CFOs**) que precisam de uma visão única e consolidada de vendas de diferentes unidades globais em tempo recorde, facilitando auditorias e avaliações de **Valuation**.



### 🛠️ Pilares de Inovação:
* **Interoperabilidade:** Utiliza **OpenFeign** para conectar sistemas legados de forma moderna e declarativa.
* **Agilidade na Tomada de Decisão:** Transforma dados brutos de APIs distintas em uma estrutura JSON padronizada para análise de performance por país.
* **Resiliência (Fault Tolerance):** O uso de padrões de projeto garante que, mesmo em ambientes de rede instáveis ou com falhas de segurança (como certificados SSL), a camada de negócio permaneça funcional através de mecanismos de *fallback* e *mocking*.

### 📈 Impacto no Negócio
Ao automatizar a consolidação de vendas, este sistema visa:
1. **Reduzir o trabalho manual** de planilhas em até **80%**.
2. **Mitigar erros humanos** em processos de integração.
3. **Entregar um Dashboard limpo** para auditorias rápidas.

---

## 🏗️ Design Patterns e Tecnologias
* **Singleton:** Instâncias únicas gerenciadas pelo Spring IoC.
* **Strategy/Service:** Centralização da lógica de negócio de forma desacoplada.
* **Proxy (OpenFeign):** Consumo de APIs externas com interface declarativa.
* **Java 17 & Spring Boot 3.5.11**.
